## 一、产品概述

### 1.1 产品定位

影院业务中心库存管理系统是一款面向多门店、多仓库场景的智能化库存管理解决方案，帮助影院企业实现库存数据的实时可视化、全流程可追溯和精细化管理。

### 1.2 核心价值

- **💡 实时可视**: 一屏掌握所有门店/仓库库存状态，库存变动实时更新
- **🔍 全程追溯**: 每一笔库存变动都有完整审计轨迹，责任可追溯
- **⚡ 高效协同**: 系统化的调拨、盘点流程，提升跨门店协作效率
- **🛡️ 风险预警**: 智能识别库存异常，及时预警低库存和超库存
- **📊 数据驱动**: 丰富的统计分析报表，支撑库存优化决策

### 1.3 适用场景

- **连锁影院集团**: 管理多个影院门店的卖品库存
- **区域仓储中心**: 统一管理区域内多个门店的库存调配
- **供应链管理**: 采购、仓储、配送全流程库存管理
- **财务对账**: 库存数据与财务系统对接，支持成本核算

### 1.4 用户角色

| 角色 | 职责 | 核心场景 |
|------|------|---------|
| 库存经理 | 全局库存管理和决策 | 库存分析、异常处理、审批调拨 |
| 仓库管理员 | 仓库日常库存管理 | 收发货、盘点、报损处理 |
| 门店管理员 | 门店库存管理 | 门店库存查看、补货申请 |
| 库存操作员 | 执行具体库存操作 | 入库、出库、扫码盘点 |
| 财务人员 | 库存财务对账 | 库存流水查询、成本分析 |

## 三、产品功能详解

### 3.1 库存查询模块

#### 📦 3.1.1 库存台账查看

**功能定位**: 实时库存状态查询和监控的核心入口

**主要功能**:

1️⃣ **多维度库存展示**
- 现存库存: 仓库实际库存数量
- 预占库存: 订单已占用但未出库的数量
- 可用库存: 可销售的库存数量 (现存 - 预占)
- 在途库存: 已采购但未到货的数量
- 安全库存: 预警阈值线

2️⃣ **智能筛选查询**
- SKU关键词搜索（支持编码、名称、拼音首字母）
- 仓库/门店多选筛选
- 商品类目层级筛选
- 品牌筛选
- 库存状态筛选（低库存/正常/超库存）
- 可售状态筛选

3️⃣ **灵活排序功能**
- 按现存库存升序/降序
- 按可用库存升序/降序
- 按在途库存升序/降序
- 按最后更新时间排序

4️⃣ **库存状态可视化**
```
🔴 低于安全库存: 可用库存 ≤ 安全库存 (红色警示)
🟢 正常库存: 安全库存 < 可用库存 ≤ 安全库存 × 3 (绿色)
🟠 超库存: 可用库存 > 安全库存 × 3 (橙色提示)
```

5️⃣ **数据导出**
- 导出当前筛选结果
- Excel格式下载
- 包含所有关键字段

**使用场景**:
- 📊 每日库存盘点: 查看各门店当前库存状态
- 🔍 异常库存定位: 快速找出低库存或超库存商品
- 📋 补货决策支持: 根据库存状态制定补货计划
- 💰 库存成本分析: 统计库存总价值

**界面特点**:
- 响应式表格设计，支持横向滚动
- 关键列固定，滚动时保持可见
- 颜色标签快速识别库存状态
- 分页加载，每页20/50/100条可选

---

#### 📝 3.1.2 库存变动日志/审计

**功能定位**: 库存变动的完整审计追踪系统

**主要功能**:

1️⃣ **变动历史追踪**
- 记录所有库存变动操作
- 包含操作时间、操作人、变动数量
- 关联业务单据（采购单、销售单等）
- 记录变动前后库存状态

2️⃣ **异常操作监控**

智能识别以下异常:
```
⚠️ 大额出库: 单次出库 > 平均值 × 3
⚠️ 频繁操作: 1小时内操作 > 50次
⚠️ 异常时间: 非工作时间操作（22:00-06:00）
⚠️ 负库存: 操作后库存 < 0
⚠️ 超量报损: 报损数量 > 库存 × 20%
```

自动触发:
- 实时预警通知
- 异常记录标记
- 触发审核流程

3️⃣ **操作人追踪**
- 查询某人的所有操作记录
- 统计操作频次和类型分布
- 识别错误操作和异常操作
- 支持绩效考核和责任追溯

4️⃣ **审计报表生成**
- 操作频次统计
- 操作人员排行
- 异常操作汇总
- 库存变动趋势分析
- 合规性审计报告

5️⃣ **多维度查询**
- 按时间范围查询
- 按操作类型筛选（入库/出库/调拨/盘点/报损）
- 按商品SKU筛选
- 按操作人筛选
- 按仓库/门店筛选

**使用场景**:
- 🔍 库存差异排查: 定位库存账实不符的原因
- 💼 财务对账: 提供完整的库存流水数据
- 🛡️ 风险审计: 识别和预防库存舞弊行为
- 📈 运营分析: 分析库存周转率和操作效率

**数据安全**:
- 审计日志不可修改、不可删除
- 保留7年（符合财务合规要求）
- 加密存储敏感数据
- 操作权限严格控制

### 3.2 库存操作模块

#### 🔄 3.2.1 库存操作管理

**功能定位**: 日常库存操作的统一入口和管理平台

**主要功能**:

1️⃣ **入库操作**

支持多种入库类型:
- 📦 采购入库: 供应商采购到货
- 🔄 调拨入库: 其他仓库调入
- 📈 盘盈入库: 盘点发现多余库存
- 🔙 退货入库: 客户退货入库

操作流程:
```
选择入库类型 → 选择仓库位置 → 扫码/输入SKU → 
输入数量 → 填写备注 → 提交确认 → 系统更新库存
```

关键特性:
- ✅ 条码扫描快速录入
- ✅ 批量入库支持
- ✅ 自动生成入库单号
- ✅ 照片凭证上传

2️⃣ **出库操作**

支持多种出库类型:
- 🛒 销售出库: 门店销售
- 🚚 调拨出库: 调往其他仓库
- 💔 报损出库: 商品损坏
- 📤 领用出库: 内部领用

操作流程:
```
选择出库类型 → 选择仓库位置 → 扫码/输入SKU → 
检查可用库存 → 输入数量 → 自动分配批次(FIFO) → 
确认出库 → 扣减库存
```

智能控制:
- 🚫 自动检查库存可用性
- 📊 先进先出(FIFO)批次分配
- ⚠️ 超量预警提示
- 🔒 权限验证控制

3️⃣ **报损操作**

报损原因分类:
- ⏰ 商品过期
- 💥 商品破损
- 🌫️ 自然损耗
- 😞 客户投诉
- 📝 其他原因

操作流程:
```
选择报损位置 → 扫码商品 → 输入报损数量 → 
选择报损原因 → 上传照片凭证 → 填写详细说明 → 
提交审批 → 审批通过后扣减库存
```

审批机制:
- 报损金额 < 500元: 自动通过
- 报损金额 500-2000元: 主管审批
- 报损金额 > 2000元: 高层审批

4️⃣ **退库操作**

退库类型:
- 🔙 销售退库: 客户退货
- 📤 采购退库: 退还供应商
- 🔄 调拨退库: 调拨商品退回

操作流程:
```
选择退库类型 → 关联原始单据 → 选择退库位置 → 
扫码商品 → 输入退库数量 → 检查商品状态 → 
确认退库 → 恢复库存
```

状态检查:
- ✅ 商品完好: 全额恢复库存
- ⚠️ 商品损坏: 记为报损，不恢复库存

**今日统计概览**:
```
📊 实时统计看板:
┌─────────────────────────────────────────┐
│ 入库单数: 12 ↑ 8.5%   出库单数: 28 ↑ 12.3% │
│ 报损单数: 3 ↓ 2.1%    退库单数: 5 ↑ 5.2%  │
└─────────────────────────────────────────┘
```

**使用场景**:
- 📦 日常收发货: 处理采购到货、门店配送
- 🔍 库存盘点: 处理盘盈盘亏
- 💔 损耗管理: 记录和处理商品损耗
- 🔄 退货处理: 处理客户退货和供应商退货

#### 🚚 3.2.2 调拨管理

**功能定位**: 跨仓库/门店的库存调配全流程管理

**调拨全流程**:
```
[调拨申请] → [审核通过] → [调出方出库] → [在途运输] → [调入方收货] → [完成]
   ↓            ↓              ↓              ↓              ↓
 待审核       待出库          在途中         待收货         已完成
```

**主要功能**:

1️⃣ **调拨申请**

申请场景:
- 🏪 门店缺货补货: 仓库 → 门店
- 🔄 门店间调拨: 门店A → 门店B
- 📦 仓库间调拨: 仓库A → 仓库B
- 🎬 活动备货: 批量调拨准备

申请流程:
```
选择调出/调入位置 → 添加调拨商品清单 → 
输入调拨数量 → 填写调拨原因 → 标记紧急程度 → 
提交审核
```

智能校验:
- ✅ 调出方可用库存检查
- ✅ 调入方是否有效
- ✅ 调拨路径合理性
- ⚠️ 紧急调拨自动优先

2️⃣ **调拨出库**

操作权限:
- 仅处理审核通过的调拨单
- 调出方仓库管理员操作

出库流程:
```
查看待出库调拨单 → 选择调拨单 → 扫码确认商品 → 
确认实际出库数量 → 打印配送单 → 确认出库 → 
扣减调出方库存 → 调拨单进入在途状态
```

关键规则:
- 实际出库数量 ≤ 申请数量
- 支持分批次出库
- 自动生成出库单号

3️⃣ **在途查询**

在途信息:
- 📍 当前位置信息
- 🚚 物流公司和单号
- ⏰ 预计到达时间
- 📊 在途时长统计
- ⚠️ 异常标记（如延误）

异常处理:
- 运输损坏记录
- 丢失报警处理
- 延误原因说明

4️⃣ **调拨收货**

收货流程:
```
查看待收货调拨单 → 选择调拨单 → 核对商品信息 → 
扫码确认收货 → 检查商品状态 → 确认收货数量 → 
增加调入方库存 → 调拨单完成
```

收货规则:
- 支持部分收货
- 差异数量记录
- 商品质量检查
- 收货凭证拍照

**使用场景**:

#### 🔒 3.2.3 库存预占/释放管理

**功能定位**: 订单库存锁定机制，防止超卖

**核心机制**:

库存计算公式:
```
可用库存 = 现存库存 - 预占库存
可售库存 = 可用库存 (前端展示)

举例:
现存库存: 100个
预占库存: 30个 (20个已支付订单 + 10个待支付订单)
可用库存: 70个 (可以继续销售)
```

**主要功能**:

1️⃣ **订单预占**

预占时机:
- 🛒 加入购物车: 软预占（5分钟）
- 📝 提交订单: 硬预占（30分钟）
- ⚡ 秒杀活动: 即时预占（10分钟）
- 💳 支付完成: 转为实际占用

预占流程:
```
检查商品库存 → 扣减可用库存 → 增加预占库存 → 
记录预占信息 → 设置过期时间 → 启动定时释放
```

预占记录:
- 订单号
- 商品SKU和数量
- 预占时间
- 过期时间
- 预占状态

2️⃣ **取消释放**

释放场景:
- 🚫 用户取消订单
- ⏰ 订单超时未支付
- ❌ 订单审核不通过
- 💥 系统异常取消

释放流程:
```
查询订单预占记录 → 恢复可用库存 → 
扣减预占库存 → 记录释放日志 → 
发送库存变动通知
```

自动释放:
- 定时任务扫描过期预占
- 每分钟执行一次
- 批量释放过期预占

3️⃣ **退款释放**

退款类型:
- 💯 全额退款: 释放所有预占
- 📊 部分退款: 按退款商品释放
- 🔄 换货不退款: 不释放库存

释放时机:
- 退货商品入库后
- 退款金额到账后
- 质检通过后

4️⃣ **预占查询**

查询维度:
- 📝 按订单号查询
- 📦 按商品SKU查询
- 🏪 按门店查询
- ⏰ 按预占时间查询
- ⚠️ 按预占状态查询

统计分析:
```
📊 预占统计看板:
┌────────────────────────────────────────────┐
│ 当前预占订单: 156                           │
│ 预占库存数量: 2,845件                       │
│ 今日释放次数: 89                            │
│ 超时待释放: 12                              │
└────────────────────────────────────────────┘

📈 预占类型分布:
订单预占: 128单 (2,450件)
活动预占: 23单 (350件)
其他预占: 5单 (45件)
```

**防超卖机制**:
```
下单前检查:
if (请求数量 > 可用库存) {
    return "库存不足，请减少购买数量"
}

预占时检查:
if (请求数量 + 当前预占 > 现存库存) {
    return "库存预占失败，库存不足"
}

并发控制:
- 使用分布式锁
- 乐观锁版本控制
- 库存扣减原子操作
```

**使用场景**:
- 🛒 在线下单: 订单提交时预占库存
- ⚡ 秒杀活动: 防止超卖
- 🎬 活动预约: 提前预留库存
- 📊 库存监控: 了解真实可售库存

### 3.3 库存分析模块

#### 📋 3.3.1 盘点模块

**功能定位**: 周期性和临时性库存盘点，确保账实相符

**盘点流程**:
```
[创建计划] → [分配任务] → [现场盘点] → [录入数据] →
```

3️⃣ **盘点审核**

差异计算:
```
盘盈数量 = 实盘数量 - 账面数量 (> 0)
盘亏数量 = 实盘数量 - 账面数量 (< 0)
盘点差异率 = |盘盈/亏数量| / 账面数量 × 100%
```

审核规则:
```
🟢 差异率 < 5%: 自动通过
🟡 差异率 5%-10%: 主管审核
🔴 差异率 > 10%: 重新盘点 或 高层审批
```

审核流程:
```
查看盘点结果 → 分析差异原因 → 
判断是否需要复盘 → 审核通过/拒绝 → 
调整库存 → 生成盘点报告
```
