### 2.2 品牌管理 · UI 设计文档（Draft）

> 目标：作为商品中心的统一品牌主数据字典，支撑 SPU / SKU / 价格 / 营销等模块引用同一套「品牌」信息。  
> 读者：前端工程师 / UI 设计师 / 产品。

---

#### 2.2.1 页面入口与布局

- 菜单路径：- 商品管理 (MDM/PIM)→ 品牌管理**
- 路由建议：`/mdm-pim/brands`
- 布局风格：标准中后台列表 + 抽屉表单

页面整体结构：

```text
┌──────────────────────────────┐
│ 顶部筛选 & 工具栏           │
├──────────────────────────────┤
│ 品牌列表表格                │
├──────────────────────────────┤
│ 分页                        │
└──────────────────────────────┘
```

- 新建 / 编辑 / 查看统一采用右侧 `Drawer` 抽屉。

---

#### 2.2.2 顶部筛选区域

**组件**：`Form`（行内） + `Input` + `Select` + `Button`

筛选项：

- 关键字（Input）
  - 占位：`输入品牌名称 / 英文名 / 编码`
- 品牌类型（Select）
  - 自有品牌 / 代理品牌 / 联营品牌 / 其他
- 所属类目（Select 或 Cascader，可选）
- 状态（Select）
  - 启用 / 停用

按钮：

- 「查询」（主按钮）
- 「重置」
- 右侧主操作：「新建品牌」

交互：

- 修改条件后点击「查询」才触发请求；
- 「重置」清空表单并自动触发一次默认查询；
- 返回列表时保留上次筛选条件。

---

#### 2.2.3 品牌列表表格

**组件**：`Table`

字段建议：

1. 品牌 LOGO（小图标，可选）
2. 品牌名称（中文）
3. 品牌英文名（可选）
4. 品牌编码
5. 品牌类型（自有 / 代理 / 联营 / 其他，Tag 展示）
6. 主营类目（饮料 / 爆米花 / 酒水 / 零食…）
7. 品牌等级（A/B/C 等，可选）
8. 状态（启用 / 停用，Tag 展示）
9. 创建时间
10. 操作列（查看 / 编辑 / 启用 / 停用）

展示规范：

- 品牌名称列可点击，打开详情抽屉；
- 状态使用不同颜色标签（启用=绿色，停用=灰/红）；
- 列表支持按创建时间排序。

---

#### 2.2.4 操作列设计

根据权限展示：

- 「查看」：打开品牌详情抽屉（只读视图）
- 「编辑」：打开编辑抽屉
- 「停用」：当状态为启用时显示
- 「启用」：当状态为停用时显示

二次确认：

- 停用时弹出确认文案：  
  > “停用后，系统中将无法在新建 SPU / SKU 时选择该品牌，历史数据不受影响。确认停用吗？”

---

#### 2.2.5 新建 / 编辑品牌抽屉

**形式**：右侧 `Drawer`，宽度约 600–720px。

表单分区：

1. 基本信息
2. 扩展信息
3. 展示信息（LOGO 等）

**字段设计：**

- 品牌名称（必填，中文）
- 品牌英文名（可选）
- 品牌编码（只读或可选，系统生成）
- 品牌类型（必选：自有 / 代理 / 联营 / 其他）
- 主营类目（Select，多选；用于筛选和分析）
- 所属公司/供应商（可选，下拉选择已有供应商）
- 品牌等级（可选：A/B/C…）
- 品牌标签（多选 Tag，如“高端”“年轻化”“进口”等）
- 品牌介绍（多行文本，用于内部说明）
- 品牌 LOGO（上传组件：单图上传，限制大小和格式）
- 状态（Radio：草稿 / 启用 / 停用；新建默认草稿）

前端校验：

- 品牌名称、品牌类型为必填；
- 品牌名称 + 品牌类型 组合建议唯一，若后端返回重复错误，在字段下提示：
  > “系统中已存在同名品牌，请检查是否重复创建。”

交互：

- 底部按钮：
  - 「保存」
  - 「取消」
- 未保存关闭抽屉时弹出确认框：
  > “当前有未保存的修改，确定要关闭吗？”

---

#### 2.2.6 品牌详情视图

展示形式：与编辑共用抽屉布局，字段全部只读。

内容结构：

1. 顶部摘要：
   - LOGO + 品牌名称（中/英）+ 状态 Tag
2. 基本信息区：
   - 编码、品牌类型、主营类目、所属公司、等级、标签
3. 文案区：
   - 品牌介绍（支持换行）
4. 关联信息入口（只读）：
   - 关联 SPU 数量（点击可跳转 SPU 列表并带筛选条件）
   - 关联 SKU 数量（同上）

顶部按钮：

- 「编辑」
- 「停用 / 启用」
- 「关闭」

---

#### 2.2.7 状态与权限控制

状态：

- 草稿：仅用于编辑阶段，不对业务开放选择（不出现在 SPU 创建下拉中）
- 启用：可被 SPU / SKU / 价格等模块选择
- 停用：不再可选，但保留历史数据

权限（前端表现）：

- 只有“主数据管理员 / 商品管理员”可以新建、编辑、停用品牌；
- 普通运营只读品牌列表与详情。

---

#### 2.2.8 设计风格补充

- 与“类目管理 / SPU 管理 / SKU 管理”保持一致的中后台视觉体系；
- 使用统一的按钮、表单、表格组件；
- LOGO 列展示为 24–32px 小缩略图，提高识别度；
- 列表为空时显示友好空态提示，并提供“新建品牌”入口。
