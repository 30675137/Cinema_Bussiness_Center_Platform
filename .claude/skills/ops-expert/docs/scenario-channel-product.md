# 典型场景：小程序新品上架

## 背景
运营小王需要在小程序上架 5 款新咖啡饮品，商品信息已在飞书表格中整理好。

---

## 第一步：创建商品管理表（首次使用）

**小王说**：`创建小程序商品管理表`

**系统执行**：
```
✅ 商品管理表创建成功

表格信息:
- 链接: https://xxx.feishu.cn/base/xxx

已创建字段: 15 个
已配置分类选项: 精品咖啡、特调饮品、甜点蛋糕...
```

---

## 第二步：填写商品数据

小王在飞书表格中录入 5 款新品：

| 商品名称 | SKU名称 | 分类名称 | 渠道价格 | 目标状态 | 同步状态 |
|---------|---------|---------|---------|---------|---------|
| 冰美式 | 美式咖啡-中杯 | 精品咖啡 | 18 | 上架 | 待同步 |
| 拿铁 | 拿铁咖啡-中杯 | 精品咖啡 | 22 | 上架 | 待同步 |
| 卡布奇诺 | 卡布奇诺-中杯 | 精品咖啡 | 24 | 草稿 | 待同步 |
| 摩卡 | 摩卡咖啡-中杯 | 精品咖啡 | 26 | 上架 | 待同步 |
| 抹茶拿铁 | 抹茶拿铁-中杯 | 特调饮品 | 28 | 上架 | 待同步 |

---

## 第三步：同步商品

**小王说**：`同步飞书商品表`

**系统执行**：
```
🔄 开始同步商品...

读取到 5 条待同步记录

处理中:
  ✅ 冰美式 - 创建成功，已上架
  ✅ 拿铁 - 创建成功，已上架
  ✅ 卡布奇诺 - 创建成功（草稿状态）
  ✅ 摩卡 - 创建成功，已上架
  ❌ 抹茶拿铁 - 失败: SKU "抹茶拿铁-中杯" 不存在

同步完成: 成功 4 条，失败 1 条
```

飞书表格自动更新：

| 商品名称 | 系统商品ID | 同步状态 | 错误信息 |
|---------|-----------|---------|---------|
| 冰美式 | prod-001 | 已同步 | |
| 拿铁 | prod-002 | 已同步 | |
| 卡布奇诺 | prod-003 | 已同步 | |
| 摩卡 | prod-004 | 已同步 | |
| 抹茶拿铁 | | 失败 | SKU "抹茶拿铁-中杯" 不存在 |

---

## 第四步：处理失败项

小王发现抹茶拿铁的 SKU 名称填错了，修正后：

| 商品名称 | SKU名称 | 同步状态 |
|---------|---------|---------|
| 抹茶拿铁 | 抹茶拿铁-标准杯 | 待同步 |

**小王说**：`同步飞书商品表`

```
✅ 抹茶拿铁 - 创建成功，已上架
```

---

## 第五步：临时下架商品

下午，摩卡原料不足需要临时下架。

**小王说**：`下架商品'摩卡'`

```
✅ 摩卡 - 已下架
```

---

## 第六步：批量上架

第二天原料到货，小王在飞书表中将摩卡的"目标状态"改为"上架"。

**小王说**：`上架待上架商品`

```
📦 批量上架商品...

找到 1 条待上架商品
  ✅ 摩卡 - 上架成功
```

---

## 第七步：查看状态

**小王说**：`查看商品同步状态`

```
📊 商品同步状态统计

飞书表格: 5 条记录，全部已同步
系统商品: 上架 5 个，草稿 0 个，下架 0 个
```

---

## 场景总结

| 步骤 | 操作 | 耗时 |
|------|------|------|
| 创建表格 | 一次性 | 10秒 |
| 录入数据 | 在飞书填表 | 人工 |
| 同步商品 | 一句话 | 5秒 |
| 上架/下架 | 一句话 | 2秒 |

**价值**：运营无需登录后台系统，在飞书表格中管理商品，通过对话完成上架下架操作。
