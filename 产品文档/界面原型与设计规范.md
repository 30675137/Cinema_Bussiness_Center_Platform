## 1. 🎨 设计原则与目标

### 1.1 核心目标

1. **统一性**：遵循一致的组件库和交互模式，确保中台各模块（商品、库存、采购）的操作体验一致。
2. **效率性**：针对高频的**商品创建、审核和价格配置**流程，优化操作路径，减少点击次数。
3. **数据治理**：通过界面设计（如强制校验提示、状态高亮），辅助运营人员保障**数据质量**。

### 1.2 关键组件规范

- **布局**：采用**左侧导航栏 + 顶部面包屑**的标准后台布局。
- **颜色**：状态颜色必须统一：**蓝色**代表**待审核/配置中**；**绿色**代表**已发布/正常**；**红色**代表**异常/驳回/缺货**。

## 2. 核心操作界面布局与规范

### 2.1 商品列表与筛选界面 (MDM/PIM)

该界面是运营人员日常工作的主要入口，需支持快速检索和状态查看。

- **布局**：顶部为**筛选区**，中部为**列表区**，右上方为**操作区**。
- **筛选区 (Search Area)**：
    - **核心筛选字段**：商品名称（模糊）、SKU ID（精确）、**类目 (Category)**（多选树形结构）、**状态 (Status)**（多选：草稿/待审核/已发布等）。
    - **物料类型筛选**：需增加**“物料类型”**下拉筛选（成品/原材料/耗材），方便运营快速管理不同类型商品。
- **列表区 (Data Grid)**：
    - **关键展示字段**：商品名称、SKU ID、**所属类目**、**物料类型**、**当前状态**、**基础价**、**实时库存**（需突出显示）。
    - **状态高亮**：`当前状态` 字段需使用颜色高亮。`待审核` 状态需有醒目的**角标提示**。
- **操作区 (Actions)**：
    - **主要按钮**：**+ 新建商品**（Primary Action）、**批量导入/导出**。
    - **行操作**：编辑、查看详情、提交审核/发布（根据当前状态动态显示）。

### 2.2 商品创建/编辑界面 (商品详情页)

采用**多 Tab + 侧边目录**的复杂表单设计，确保信息完整性和逻辑清晰。

| **Tab/区块名称** | **核心字段/功能** | **交互规范重点** |
| --- | --- | --- |
| **基础信息** (Core MDM) | **类目选择**（必选）、**物料类型**（类目联动显示）、SPU/SKU 名称、条码、品牌、单位。 | **强制校验**：类目选择后，其**物料类型**不可修改。必填字段红色星号提示。 |
| **内容管理** (PIM) | 标准标题、长/短描述、图文详情、视频素材上传。 | **渠道覆写入口**：设置一个**“渠道内容配置”**按钮，进入二级浮窗，配置小程序短标题等**覆写字段**。 |
| **规格与属性** | 规格项（如颜色、尺码）配置、属性值录入。 | **表格联动**：配置规格项后，底部自动生成 **SKU 矩阵**，支持批量填写价格和条码。 |
| **BOM/配方** | **BOM Header** 配置、**BOM Detail** 列表。 | **逻辑控制**：只有**物料类型**为**成品**的商品才显示此 Tab。配方组件必须从**原材料**列表中选择。 |

### 2.3 价格配置界面 (定价中心)

该界面用于维护门店/渠道的价格关系，需支持批量操作。

- **定位**：独立于商品详情页，入口位于 **“定价中心”** 模块。
- **配置方式**：采用**配置单**模式，而非直接修改数据库。
- **核心功能**：
    1. **选择维度**：选择 **SKU 范围**、**门店群组/渠道**。
    2. **录入价格**：录入**基础价**、**时段价规则 ID**、**会员价规则 ID**。
    3. **价格预览 (Preview)**：提供一个**“价格裁决预览”**按钮，运营可查看配置后，任意 SKU 在任意门店的最终价格口径，再进行发布。
    4. **发布/生效**：配置单提交后，需经过**审核**，并执行**价格关系表 (Price_Store_Relation)** 的更新。

### 2.4 审核界面 (Pending Review)

此界面是保障数据质量的关键，需要强制运营人员关注修改点。

- **布局**：简洁，左侧为**商品信息概览**，右侧为**修改对比/审核意见区**。
- **核心交互**：
    1. **高亮对比**：系统自动识别并**高亮显示** `Draft` 状态下所有**被修改过的核心字段**（标题、价格、配方组件、关键属性）。
    2. **强制确认**：审核员必须点击所有被高亮修改的字段旁的**“已确认”**按钮，才能激活底部的**“审核通过”**按钮。
    3. **意见与驳回**：提供**“驳回”**按钮，强制要求填写驳回原因，驳回后商品状态回到 `Draft`。

### 2.5 库存追溯界面 (Inventory Tracking)

该界面用于解决**库存准确性**问题，支持对所有交易流水进行追溯。

- **功能布局**：
    1. **顶部查询**：SKU ID（必填）、门店/仓库 ID、**交易类型 (Trans_Type)**（如：采购入库、销售出库、盘点）筛选。
    2. **实时台账展示**：在顶部小卡片实时显示该 SKU 在该门店的 `Available_Qty`、`On_Hand_Qty`。
    3. **交易流水列表**：按时间倒序展示 `Inventory_Transaction` 表的所有记录。
    4. **流水详情**：每条流水需展示：**变动数量**、**变动前/后数量**、**批次成本 (Batch_Cost)**、**来源单据 ID**（如 PO ID、订单 ID）。
